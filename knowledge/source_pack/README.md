# Source-pack database

`source_database.json` and `source_database.csv` are generated by
`tools/extract_source_pack.py`. Every row includes its source file,
page/image, section, confidence, extraction method, review state, notes, and
an `ignore_for_dps` flag.

Data policy:

- `exact` rows may feed action templates only when `needs_review=false`.
- `inferred`, `opinion`, and `placeholder` rows never override exact rows.
- Raw image OCR is retained as `raw_evidence` with `needs_review=true`; its
  numbers are not assigned a game meaning without verification.
- Survival-only facts are retained with `ignore_for_dps=true`.
- `action_templates.json` contains compact executable templates. Profile
  ownership, progression, and resource gates are evaluated at runtime.
- `../gpu_tables/source_pack/numeric_tables.json` contains stable integer ID maps and rectangular
  numeric arrays for actions, resources, unlocks, breakpoints, and damage.

Regenerate and perform the fast semantic check:

```powershell
.\.venv\Scripts\python.exe tools\extract_source_pack.py
.\.venv\Scripts\python.exe tools\validate_source_pack.py
```

The review backlog is in `../review_queue/source_pack_queue.jsonl`.
Validation reports are written under `reports/`. The validation report clearly
separates represented systems from systems that still need exact normalization.
